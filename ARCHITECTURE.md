## Non-Functional Requirements

### Scalability
- **Horizontal scaling**: Stateless API design allows multiple backend instances
- **Database pooling**: Prisma manages connection pools efficiently
- **Message pagination**: Chat history limited to last 50 messages for context

### Security
- **Password hashing**: bcrypt with 12 salt rounds
- **JWT tokens**: 7-day expiry, signed with secret key
- **Input validation**: All inputs validated server-side
- **SQL injection**: Prevented via Prisma parameterized queries
- **CORS**: Configured for specific frontend origin

### Extensibility
- **Modular routes**: Easy to add new API endpoints
- **Plugin-ready**: LLM provider can be swapped (OpenAI, Anthropic, etc.)
- **File storage**: Abstracted for cloud storage integration

### Performance
- **Database indexes**: Primary keys and foreign keys indexed
- **Response streaming**: SSE endpoint for real-time chat
- **Optimistic UI**: Frontend shows messages before confirmation

### Reliability
- **Error handling**: Global error middleware catches all errors
- **Graceful degradation**: AI errors return user-friendly messages
- **Input validation**: Prevents malformed requests

## Technology Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Backend Framework | Express.js | Minimal, fast, large ecosystem |
| Database | PostgreSQL | ACID compliance, scalability |
| ORM | Prisma | Type safety, migrations, DX |
| Auth | JWT | Stateless, scalable |
| LLM API | OpenRouter | Free tier, multiple models |
| Frontend | React + Vite | Fast DX, modern tooling |
| Styling | Tailwind CSS | Rapid development, consistency |
